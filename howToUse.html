Синтаксис медіа-запитiв

Запис медіа-запитiв на сайті виглядає ось так:
@media умова { /* стилі, які стають активними за умови }
Можна додавати їх у HTML-частини сайту, всередині таблиць CSS, а також напрямок у коді сторінки. Якщо мова іде про старі браузери, то можна підключити спеціальну JavaScript-бібліотеку, щоб медіа-запити коректно працювали.

Головні види пристроїв, які можна записати в медіа-запитах:

– Всi пристрої - all;
– Пристрої з екранами - screen;
– Cинтезатори мови - speach;
– Принтери або сторінка попереднього перегляду для друку - print.

Логiчнi оператори

Також в медiа-запитах можуть утримувати в собi логiчнi оператори:
– and - вci умови повиннi виконувутися одночасно;
– , - хоча б одна умова повинна бути виконана;
– not - заперечення умови (має нижчий пріоритет, ніж and, отже виконуватиметься після нього).

Функцiї

– width (max-width та min-width) — визначає вимоги до ширини браузера, найбільше та найменше значення цього параметра в пікселях або інших одиницях: @media(width: 720px);
– height, min-height і max-height — прописує вимоги до висоті браузера, аналогічно до функцій ширини: @media (max -height: 1080px);
– color визначає кількість біт на канал кольору (також є значення min та max);
– aspect-ratio (min-aspect-ratio, max-aspect-ratio) — відношення ширини до висоти: @media screen and (device-aspect-ratio: 1920/1080);
– orientation — оцінює режим відображення сторінки: портретний (portrait) або альбомний (landscape);
– resolution (min-resolution, max-resolution) — це дозвіл пристрою введення: необхідний, мінімальний та максимальний; вимірюється в dpi, dpcm чи dppx.

Які бувають медіа-запити

Медіа-запити можуть записуватися як атрибут для елемента link. Це допомагає визначити, які файли CSS-таблиць потрібно активувати на сторінці в залежності від параметрів пристрою. Такий функціонал найчастіше використовується, коли до різних типів технічного обладнання використовуються різні стилі CSS.
Наведемо приклад фрагмента коду, який ілюструє підключення файлу styles-xs.css для пристроїв, розмір екрану яких менше 543 пікселів:
link rel="stylesheet" media="screen and (max-width: 543px)" href="styles-xs.css"
Також подібні запити можна включати до складу правила @import, що відповідає за імпорт стилів з інших файлів CSS. Так, для того ж файлу можна записати наступний код:
@import url(styles-xs.css) (max-width: 543px);
Медіа-запити можна реалізувати для Bootstrap 3 та Bootstrap 4. Наприклад, можна прописати media queries у порядку збільшення розмірів для кількох класів пристроїв:
@media (min-width: 768px) {/* Стилі для пристроїв з маленьким екраном, розмірами 768px - 991px (планшети та подібні) */}
@media (min-width: 992px) {/* Стилі для ноутбуків та комп'ютерів з екранами в діапазоні 992px – 1199px */}
@media (min-width: 1200px) {/* Стилі для пристроїв із великою шириною екрану >1200px */}
Використовувати ці медіа-запити потрібно виключно у такому порядку. Якщо ви хочете застосовувати їх у довільній послідовності, додайте в них максимальне значення. Тобто вкажіть повноцінний діапазон розмірів екранів у Bootstrap 3:
@media (max-width: 767px) {...}
@media (min-width: 768px) and (max-width: 991px) {...}
@media (min-width: 991px) and (max-width: 1199px) {...}
@media (min-width: 1200px) {...}
Якщо мова йде про медiа-запити в Bootsrap 4, то можна написати таке поступове збільшення найменшої ширини екрана:
@media (min-width: 576px) {/* CSS для екранiв розмiрами: 576px – 767px */}
@media (min-width: 768px) {/* CSS для: 768px – 991px */}
@media (min-width: 992px) {/* CSS для: 992px – 1119px */}
@media (min-width: 1200px) {/* CSS для екранiв бiльше 1200px */}
Подiбнi запити, але з максимальним значенням можна застосовувати у зворотному порядку, щоб йшло спадання розмірів екрана. Якщо ви хочете писати медіа-запити у випадковiй послідовності в таблиці стилів, то вкажіть точні проміжки, тобто одразу пропишіть min-width и max-width. Тоді фрагмент коду виглядатиме ось так:
@media (max-width: 575px) {...}
@media (min-width: 576px) and (max-width: 767px) {...}
@media (min-width: 768px) and (max-width: 991px) {...}
@media (min-width: 992px) and (max-width: 1199px) {...}
@media (min-width: 1200px) {...}

Стратегії використання медіа-запитiв

Для створення дизайну, що дозволяє найкраще відображати сайт на різних пристроях, використовують загальні стратегії медіа-запитів:
• Зменшення кількості колонок (стовпців) та поступове скасування обтікання для мобільних пристроїв;
• Використання властивості max-width замість width за завдання ширини блока-контейнера;
• Зменшення полів та відступів на мобільних пристроях (наприклад, нижніх відступів між заголовком та текстом, лівого відступу для списків тощо);
• Зменшення розмірів шрифтів для мобільних пристроїв;
• Перетворення лінійних навігаційних меню в розкривні;
• Приховування другорядного вмісту на мобільних пристроях за допомогою display: none;
• Підключення фонових зображень зменшених розмірів.

Практика створення адаптивного макету

Уявимо, що нам потрібно зробити адаптивний макет, розберемо як це зробити за кроками:
Крок №1 – створюємо html файл Формуємо скелет майбутньої html сторінки
<div class="container">
<main class="main">
MAIN
</main>
<aside class="aside">
ASIDE
</aside>
</div>
 Крок №2 – додаємо тег ><style> На початок створеного html файлу додаємо тег<style> для визначення стилів компонентів сторінки. Робимо відступ 0 і додаємо гнучкий контейнер (display: flex та flex-wrap: wrap). Потім вказуємо ширину та мінімальну висоту (у пікселях) для елементів aside та main:
body {
margin: 0;
}
.container {
display: flex;
flex-wrap: wrap;
}
.aside, .main {
width: 100%;
min-height: 1px;
}
 Крок №3 – формуємо розміри блоків для хв. дозволу 990px Вводимо перший медіа-запит, щоб вказати мінімальну ширину елемента (990px) та додати параметри aside (flex: 33,3% або <=990px) та main (flex: 66,6%):
@media (min-width: 990px) {
.aside {
flex: 0 0 33.333333%;
max-width: 33.333333%;
order: 1;
}
.main {
flex: 0 0 66.666667%;
max-width: 66.666667%;
order: 2;
}
}
 Крок №4 – формуємо розміри блоків для хв. дозволу 1400px Вводимо другий медіа-запит – вказуємо мінімальну ширину елемента 1400px і аналогічно прописуємо aside (flex і max-width: 25%) та main (flex і max-width: 75%):
@media (min-width: 1400px) {
.aside {
flex: 0 0 25%;
max-width: 25%;
}
.main {
flex: 0 0 75%;
max-width: 75%;
}
}
 Крок №5 – підсумковий html файл У підсумку, цілий код може виглядати так:
<style>
body {
margin: 0;
}
.container {
display: flex;
flex-wrap: wrap;
}
.aside, .main {
width: 100%;
min-height: 1px;
}
@media (min-width: 992px) {
.aside {
flex: 0 0 33.333333%;
max-width: 33.333333%;
order: 1;
}
.main {
flex: 0 0 66.666667%;
max-width: 66.666667%;
order: 2;
}
}
@media (min-width: 1400px) {
.aside {
flex: 0 0 25%;
max-width: 25%;
}
.main {
flex: 0 0 75%;
max-width: 75%;
}
}
</style>

<main class="main">
MAIN
</main>
<aside class="aside">
ASIDE
</aside>
</div>

Приклад

Щоб вищезазначені правила краще засвоїлися, пропонуємо ознайомитися з кодом, який використовується для встановлення ширини за правилами адаптивної верстки:
@media all (min-width: 1200px) {
div {
width: 800px
}
}
@media all (min-width: 400px) {
div {
width: 350px
}
}
 А цей код вказує ширину та висоту сторінки:
@media all (min-width: 1200px) {
div {
width: 800px
}
}
@media all (min-width: 400px) {
div {
width: 350px
}
}
 Щільність пікселів, орієнтація сторінки задаються так само.

Заключення

Медіа-запити є важливим елементом створення оптимізованого дизайну сайту. Вони керують стилями, орієнтуючись на технічні характеристики пристроїв. Це означає, що CSS-таблиці активуватимуться лише після визначення того, де саме запускається ресурс. У загальному вигляді медіа-запит складатиметься з ключового слова, що оголошує тип пристрою, а також вирази для перевірки його характеристик.
