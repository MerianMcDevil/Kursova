Що таке медiа запити в CSS

Медіа-запити — це особливі правила застосування CSS, за допомогою яких можна змінити елементи стилю, в залежностi від пристроїв, які будуть розглядати веб-сторiнку. Простiше кажучи, вони дозволяють зрозуміти, які саме стилі потрібно запустити на обранiй сторінці, а які — не треба. Наприклад, з ними сайт однаково зручно і гарно відображається на екрані смартфону або великому моніторі. Медіа-запити стали актуальними в CSS3. На сьогодні вони підтримуються всіма сучасними браузерами, серед яких Chrome, Opera, Firefox, Safari. Ключова задача медiа-запитiв — адаптивний дизайн сайту. Завдяки їм зовнiшнiй вигляд сторiнок буде варіюватися, виходячи з розмірів монітора чи екрана.

Таке поняття, як медіа-запити, з'явилося у CSS3. З їх допомогою можна визначати зовнішній вигляд веб-сторінок, спираючись на ширину та висоту вікна браузера. Завдяки медіа-запитам ви можете паралельно створювати стилі для невеликих мобільних телефонів, планшетів, лептопів та моніторів із великою діагоналлю.

Ідея адаптивного дизайну – надати відвідувачам сайт у найбільш зручній для читання та привабливій формі. Дизайн створюється під як мінімум три цільові екрани: настільний ПК, планшет і смартфон. Але враховуючи велику кількість варіацій цих девайсів, недостатньо зупинитися на одній ширині для кожного типу пристрою. Тому важливо використовувати гнучку сітку та проводити тестування у процесі розробки, відстежуючи те, як підлаштовується дизайн сайту під різну ширину браузерного вікна.

Коли використовувати медіа-запити?


Як правило, для адаптації дизайну веб-сторінки необхідно прописати ряд стилів CSS для певних елементів. Нижче ми перерахуємо найчастіші сценарії, коли вам знадобиться звернутися до медіа-запитів.
• Визначення кількості колонок. Триколонковий макет сайту, який шикарно виглядає на настільних ПК, буде зовсім недоречний для мобільних телефонів та планшетів. За допомогою медіа-запитів можна перевизначити кількість колонок у макеті з урахуванням ширини екрана пристрою, згрупувавши весь контент для смартфонів в одну колонку.
• Відносна ширина. Коли ви встановлюєте фіксовану ширину макета, скажімо, 960 пікселів, це підійде для настільних комп'ютерів, лептопів і деяких особливо великих планшетів. Однак, для мобільного телефону такий розмір контейнера занадто великий. Використовуючи медіа-запит, можна спеціально для смартфонів створити гумовий макет, ширина якого буде не фіксованою, а відносною. Таким чином, контейнер шириною 100% буде легко підлаштовуватись під будь-який смартфон у будь-якій орієнтації.
• Зменшення відступів. Великі відстані між блоками та елементами створюють відчуття повітряного, легкого дизайну. Однак відступи, які добре виглядають на моніторі з діагоналлю 21-27 дюймів, будуть виглядати абсолютно потворно на невеликому екрані смартфона, примушуючи користувачів більше прокручувати сторінку. Медіа-запити дозволяють встановити альтернативні розміри відступів для вузьких дисплеїв.
• Налаштування розміру шрифту. Заголовки висотою 60 пікселів можуть красиво виглядати на десктопі, але для мобільних пристроїв такий розмір шрифту рідко коли доречний, тому для вузьких екранів буде доцільним встановити менше значення font-size.
• Адаптивна навігація. Нерідко буває так, що навігаційне меню, яке було продумано для десктопної версії сайту, зовсім не підходить для мобільних пристроїв через свою об'ємність або манери розташування пунктів меню. Існує кілька підходів до реалізації зручної навігації для мобільної версії сайту, і медіа-запити неодмінно беруть участь у цьому.
• Приховування елементів. Деякі частини веб-сторінки можуть бути марними або незначними, якщо сайт переглядається з мобільного телефону. За допомогою властивості display: none та медіа-запитів ви можете приховати певні елементи від очей відвідувача, якщо він зайшов на сайт зі смартфона. Але майте на увазі, що подібний прийом не заощаджує трафік: приховані елементи, як і раніше, завантажуватимуться браузером.
Звичайно, це далеко не всі ситуації, коли використовуються медіа-запити.
